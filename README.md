## 📌 Sobre o repositório

Este repositório é um **fork pessoal** da trilha oficial de Python da [Digital Innovation One (DIO)](https://www.dio.me/), usado para registrar meus estudos, exercícios práticos e desafios de código conforme avanço pelos módulos da formação.

O objetivo aqui não é só "fazer o curso", mas construir um histórico real de evolução: cada pasta representa um módulo da trilha, com os conceitos e exercícios trabalhados em Python — da base da linguagem até o desenvolvimento de APIs com frameworks modernos (Flask, Django e FastAPI).

> 🔗 Repositório original: [`digitalinnovationone/trilha-python-dio`](https://github.com/digitalinnovationone/trilha-python-dio)

## 🗂️ Estrutura da trilha

A trilha é dividida em módulos numerados. Abaixo, a estrutura real encontrada neste fork e o que cada módulo aborda:

| # | Módulo | Conteúdo abordado |
|---|--------|--------------------|
| `00` | **Fundamentos** | Variáveis, tipos de dados, operadores, estruturas condicionais e de repetição, funções, entrada e saída de dados |
| `01` | **Estrutura de dados** | Listas, tuplas, dicionários, sets, compreensões de lista (*list comprehensions*) |
| `02` | **Programação Orientada a Objetos** | Classes, objetos, atributos, métodos, herança, encapsulamento e polimorfismo |
| `03` | **Decoradores, Iteradores e Geradores** | Decorators, iteradores personalizados, generators e expressões geradoras |
| `04` | **Data e hora** | Módulo `datetime`, manipulação de datas, fusos horários e formatação temporal |
| `05` | **Manipulação de arquivos** | Leitura e escrita de arquivos `.txt`, `.csv` e `.json` |
| `06` | **Gerenciamento de pacotes e boas práticas** | `pip`, ambientes virtuais, PEP 8 e boas práticas de código Python |
| `07` | **Banco de dados** | Conexão com SQLite, execução de queries e manipulação de dados via Python |
| `11` | **Desenvolvimento de APIs com Flask** | Criação de APIs REST, rotas, métodos HTTP e integração com banco de dados |
| `12` | **Desenvolvimento fullstack com Django** | Arquitetura MVC, templates, ORM e administração via Django Admin |
| `13` | **APIs Assíncronas com FastAPI** | Endpoints assíncronos, validação de dados com Pydantic e documentação automática |

## 🚀 Como executar

```bash
# Clone este repositório
git clone https://github.com/YugoPereira/trilha-python-dio.git

# Acesse a pasta
cd trilha-python-dio

# Acesse o módulo desejado
cd "01 - Estrutura de dados"

# (Opcional) Crie e ative um ambiente virtual
python -m venv venv
source venv/bin/activate      # Linux/Mac
venv\Scripts\activate         # Windows

# Execute o arquivo desejado
python nome_do_arquivo.py
```

> 💡 Para os módulos de Flask, Django e FastAPI, verifique se há um `requirements.txt` na respectiva pasta e instale as dependências com `pip install -r requirements.txt` antes de rodar o servidor local.
